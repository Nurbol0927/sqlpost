Экзаменге Даярдык

1. VCS деген эмне? Ал эмне үчүн чыгарылган?
VCS (Version Control System) – бул файлдардын жана коддун өзгөрүүлөрүн көзөмөлдөөчү система. Бул командалык иштешүүнү жакшыртуу жана мурунку версияларга кайтып келүү үчүн колдонулат.
2. Git деген эмне?
Git – бул таратылган версияны башкаруу системасы, ал коддун өзгөрүүлөрүн көзөмөлдөйт жана иштеп чыгуучуларга биргелешип иштөөгө жардам берет.
3. GitHub деген эмне?
GitHub – бул Git репозиторийлерин сактоо жана алар менен иштөө үчүн булуттык платформа.
4. Репозиторий деген эмне?
Репозиторий – бул Git менен башкарылуучу папка, анда бардык файлдар жана алардын өзгөрүүлөрү сакталат.
5. Branch деген эмне?
Branch (ветка) – бул негизги коддон бөлүнүп, өз алдынча иштөөгө мүмкүнчүлүк берген өзгөрүүлөрдүн тармагы.
6. Master ветка деген эмне?
Master – бул долбоордун негизги жана туруктуу версиясы сакталуучу ветка.
7. Owner деген ким?
Owner – бул GitHub репозиторийинин ээси, ал бардык укуктарга ээ.
8. Collaborator деген ким?
Collaborator – бул репозиторий менен иштөөгө уруксат алган колдонуучу.
9. Git тин кандай командалары бар?
  • git init – жаңы репозиторий түзөт.
  • git clone – бар репозиторийди көчүрөт.
  • git add – өзгөртүлгөн файлдарды stage кылат.
  • git commit – өзгөрүүлөрдү сактайт.
  • git push – өзгөрүүлөрдү алыстагы репозиторийге жиберет.
  • git pull – алыстагы өзгөрүүлөрдү жүктөп алат.
10. БД деген эмне?
База данных (БД) – бул структураланган маалыматтар топтому, аларды сактоо жана башкаруу үчүн колдонулат.
11. СУБД деген эмне?
Система управления базами данных (СУБД) – бул маалыматтарды башкаруучу программа (PostgreSQL, MySQL, Oracle ж.б.).
12. СУБД менен РСУБД ның айырмасы?
Реляциялык СУБД (РСУБД) – бул маалыматтарды таблицалар түрүндө сактаган СУБД (PostgreSQL, MySQL ж.б.).
13. PostgreSQL деген эмне?
PostgreSQL – бул ачык булактуу реляциялык маалымат базасынын башкаруу системасы.
14. SQL деген эмне?
SQL (Structured Query Language) – бул маалымат базасын башкаруу үчүн колдонулган тил.
15. SQL кандай болуп бөлүнөт?
SQL негизги 3 бөлүккө бөлүнөт: DDL (маалымат структурасын түзүү), DML (маалымат менен иштөө), DCL (укуктарды башкаруу).
16. DDL жана DML деген эмне?
DDL (Data Definition Language): CREATE, ALTER, DROP. DML (Data Manipulation Language): INSERT, UPDATE, DELETE.
17. Data Types (varchar, numeric, serial ж.б.)
PostgreSQL маалымат түрлөрүнө varchar, numeric, integer, serial ж.б. кирет.
18. Primary Key жана Foreign Key эмне?
Primary Key – таблицанын уникалдуу идентификатору, Foreign Key – башка таблицага байланышы.
19. Кандай Constraints (ограничение) бар?
NOT NULL, UNIQUE, PRIMARY KEY, FOREIGN KEY, CHECK, DEFAULT.
20. Join деген эмне?
JOIN – бул SQLде эки же андан көп таблицаларды бириктирүү операциясы.
21. Join дын түрлөрү
INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL JOIN, CROSS JOIN.
22. Подзапрос деген эмне?
Подзапрос – бул башка SQL суроосунун ичинде колдонулган суроо.
23. Аггрегатные функции деген эмне?
SUM, COUNT, AVG, MAX, MIN – бул функциялар маалыматтардын жыйынтыктарын эсептөө үчүн колдонулат.
